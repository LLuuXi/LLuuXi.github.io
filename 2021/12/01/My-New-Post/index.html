草地生成工作流

发表于2021-12-01|更新于2021-12-06|HDRP卡通渲染
|阅读量:

序言

最近项目中需要为场景添加草地，开发了一段时间。记录在此过程中的一些心得。这篇文章包含了：shader，草地工具，数据优化，内置管线->HDRP迁移，一些之后需要改进的方向

Mesh

   一般来说草,花长什么样子。应该按照自然界中的真实形状来处理，在此基础上做一些风格化的修改。这本是理想状态。但在游戏开发中，这些地表植被是比较大量出现的。往往占据了屏幕的大部分像素。所以归根结底还是硬件问题，不管是移动端还是桌面端我们最后还是得考虑同屏数据量，Overdraw,批次等问题。GPU精粹系列其中有一章节(原文链接)是关于草的它里面的Mesh如下：

远处无需过多细节的时候lod成广告牌的模式，塞尔达荒野之息里面采用类似的方案。

近处多细节的时候，采用这种米字型的设计，可以在较少面数的情况下，做到视角环绕一周也能有不错的细节覆盖。当然从Top视图看的时候往往能发现这种视觉上的trick，法线的处理也是采用统一向上，尽可能让光照变得更加统一自然而不随着地表的坡度有着明显的变化，
从原神的截图：

开始制作资源时其实我自己还是比较喜欢育碧<渡神纪 芬尼斯崛起>里面的做法，草的整体是比较笔直的这个风吹的时候看上去很柔顺，大多数采用3段式建模，边上带点2段式的。顶上带一点特殊的形状，这样能够带来帮更加出色的视觉感受。截图如下：

在内置管线的项目中是一款移动端的手游，且美术希望更像荒野之息里面的表现。截图如下：


法线的处理跟GUPGems里的做法一致，刷了顶点色控制软硬。
   项目中采用Graphics.DrawMeshInstanced绘制值草地，API单次最多能填充1023个数据，基于对每个chunk的大小，和草之间的间距，做了相应的规范，实际上chunk并没有满载。这样可能会带来额外的DC,不过问题不大。最后在做些特殊需求的时候可能会遇到更为棘手的问题，如垂直方向的多层的植被分布，先挖个坑以后再来补上。

Shader

到了新项目，从内置管线迭代到了HDRP，对比前后的更改

vertex部分
//草地自身风吹草动的计算
float4 worldPos = mul(unity_ObjectToWorld, v.vertex);
float2 samplePos = worldPos.xz / _WaveControl.w;
samplePos += _Time.x * -_WaveControl.xz;
fixed waveSample = tex2Dlod(_Noise, float4(samplePos, 0, 0)).r;
o.tempCol = waveSample;
worldPos.x += sin(waveSample * _WindControl.x) * _WaveControl.x * _WindControl.w * v.uv.y;
worldPos.z += sin(waveSample * _WindControl.z) * _WaveControl.z * _WindControl.w * v.uv.y;
//草地交互的计算
float dis = distance(_PlayerPos, worldPos);
float pushDown = saturate((1 - dis + _PushRadius) * v.uv.y * _Strength);
float3 direction = normalize(worldPos.xyz - _PlayerPos.xyz);
direction.y *= 0.5;
worldPos.xyz += direction * pushDown;

fragment部分
float3 color = lerp(_BottomColor_DIY,_TopColor_DIY2,i.uv.y).rgb;
