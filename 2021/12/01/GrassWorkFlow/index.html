草地生成工作流

发表于2021-12-01|更新于2021-12-07|HDRP卡通渲染
|阅读量:

序言

最近项目中需要为场景添加草地，开发了一段时间。记录在此过程中的一些心得。这篇文章包含了：shader，草地工具，数据优化，内置管线->HDRP迁移，一些之后需要改进的方向

Mesh

   一般来说草,花长什么样子。应该按照自然界中的真实形状来处理，在此基础上做一些风格化的修改。这本是理想状态。但在游戏开发中，这些地表植被是比较大量出现的。往往占据了屏幕的大部分像素。所以归根结底还是硬件问题，不管是移动端还是桌面端我们最后还是得考虑同屏数据量，Overdraw,批次等问题。GPU精粹系列其中有一章节(原文链接)是关于草的它里面的Mesh如下：

远处无需过多细节的时候lod成广告牌的模式，塞尔达荒野之息里面采用类似的方案。

近处多细节的时候，采用这种米字型的设计，可以在较少面数的情况下，做到视角环绕一周也能有不错的细节覆盖。当然从Top视图看的时候往往能发现这种视觉上的trick，法线的处理也是采用统一向上，尽可能让光照变得更加统一自然而不随着地表的坡度有着明显的变化。
开始制作资源时其实我自己还是比较喜欢育碧<渡神纪 芬尼斯崛起>里面的做法，草的整体是比较笔直的这个风吹的时候看上去很柔顺，大多数采用3段式建模，边上带点2段式的。顶上带一点特殊的形状，这样能够带来帮更加出色的视觉感受。截图如下：

在内置管线的项目中是一款移动端的手游，且美术希望更像荒野之息里面的表现。截图如下：


法线的处理跟GUPGems里的做法一致，刷了顶点色控制软硬。
   项目中采用Graphics.DrawMeshInstanced绘制值草地，API单次最多能填充1023个数据，基于对每个chunk的大小，和草之间的间距，做了相应的规范，实际上chunk并没有满载。这样可能会带来额外的DC,不过问题不大。最后在做些特殊需求的时候可能会遇到更为棘手的问题，如垂直方向的多层的植被分布，先挖个坑以后再来补上。

Shader

vertex部分
//草地广告牌
float3 viewer = mul(unity_WorldToObject, float4(_WorldSpaceCameraPos, 1));
float3 normalDir = viewer;
normalDir.y = 0;
normalDir = normalize(normalDir);
float3 upDir = float3(0, 1, 0);
float3 rightDir = normalize(cross(upDir, normalDir));
upDir = normalize(cross(normalDir, rightDir));
o.uv = v.uv.xy;
o.vertexColor = v.vertexColor;
float4 localPos = float4( rightDir * v.vertex.x + upDir * v.vertex.y + normalDir * v.vertex.z,1);
v.vertex.xyz = localPos.xyz;

//风
half windScope = _WindScope;
half4 windParameter = _WindParameter;
half4 softness = _Softness;
float4 worldPos = mul(unity_ObjectToWorld, v.vertex);
float2 samplePos = worldPos.xz / windScope;
samplePos += _Time.x * windParameter.xy;
half windMask = tex2Dlod(_VertexAnimMap, float4(samplePos,0,0)).r;
windParameter.xy = normalize(windParameter.xy);
half softnessFactor = (1 - v.vertexColor.a) * softness * 0.05;
worldPos.x += windMask * windParameter.z * windParameter.x * softnessFactor;
worldPos.z += windMask * windParameter.w * windParameter.y * softnessFactor;
//worldPos.y -= v.uv.y * (1 - v.vertexColor.a) * length(windParameter.xz + displacement.xy) * 0.5f;
v.vertex.xyz = mul(unity_WorldToObject,worldPos).xyz;

float waveSample = 0;
float2 gust = 0;
float randpos = 0;
float pushDown = 0;
if (viewDistanceLimit > 0)
{
float softness = input.uv0.y;
float2 GustingUV = (AbsolutePositionWS.xz * (0.008 + _WindGlobal2.x * 0.0001)) + _TimeParameters.x * float2(0.02, 0.02) * _WindGlobal2.y;
gust = (SAMPLE_TEXTURE2D_LOD(_MixMap2, sampler_MixMap2, GustingUV, 0).rg * 2 - 1) * softness * softness;
gust = min(gust, 0.5);
float4 offset;
offset.xz = (gust * _WindGust) * viewDistanceLimit;
offset.y = softness;
float windWeight = length(offset.xz) + 0.0001;
windWeight = pow(windWeight, 1.5);
offset.y *= windWeight;
positionRWS.xz -= offset.xz;
positionRWS.y -= offset.y;
//草地交互的计算
float playerDis = distance(_PlayerPos.xz, GetAbsolutePositionWS(positionRWS).xz);
float pushDown = saturate((1 - playerDis - 0.2) * (input.uv0.y * _GrassInteraction));
offset.xyz = -normalize(GetAbsolutePositionWS(positionRWS) - _PlayerPos) * pushDown;
offset.w = softness * 2;
windWeight = length(offset.xyz) + 0.0001;
windWeight = pow(windWeight, 1.5);
offset.w *= windWeight;
positionRWS.xz -= offset.xz;
positionRWS.y -= offset.w;
}

float3 specularLighting = lightCol * shadowCol * lightWrapData.specularDimmer * specR * disFactor;
float3 specMask = shadowCol * lightWrapData.specularDimmer * specR * disFactor * sssFactor * 5;
lighting.specular = saturate(specularLighting * sssFactor * 0.5);
half3 sssDir = L - N;
half sssVdotL = pow(saturate(dot(V, -sssDir)), 5);
float backSSS = sssVdotL * sssFactor * 0.5;
float diffTerm = DisneyDiffuse(clampedNdotV, abs(celNdotL), LdotV, bsdfData.perceptualRoughness);
float diffR = diffTerm * clampedNdotL;
float3 hsvCol = RgbToHsv(bsdfData.diffuseColor);
hsvCol.z = saturate(specMask.r + hsvCol.z);
bsdfData.diffuseColor = HsvToRgb(hsvCol);
float3 diffuseLighting = bsdfData.diffuseColor * lightCol * shadowCol * diffTerm * lightWrapData.diffuseDimmer;

刷草工具

主要功能界面如下(内置管线版本，HDRP管线做了一些变动)
开始设计的时候最要参考了一些Unity原生地形的设计。然后考虑设计4个模块：

  1. 资源选择预览
  2. 刷草，刷颜色
  3. 数据保存，优化
  4. 编辑器配置

资源选择预览：

为了方便预览添加的资源长啥呀，用到了AssetPreview.GetAssetPreview这个API,凡是添加到这里面的资源依旧会保存到编辑器的配置记录里面，下次打开依旧存在。
草地编辑：

  1. 生成笔刷范围类的种子。种子位置可以有序，无序按需求来就行。

  2. 利用是生成好的种子数据，生成射线，然后射线对植被覆着物进行检测，获取满足要求的覆着物，然后记录各种信息如：法线，uv，纹理等。
    部分代码： 对于地表来说行走区域等特殊区域边缘地区需要植被相对稀疏或者种类发生变化（变为车前草之类的抗压植物）。

    颜色编辑：

    主要是针对已经刷好的草地，根据笔刷的中心位置到笔刷边的缘渐变来刷颜色，其余染色桶，HSV的调整都是简单的色彩变化。用来处理没有实时光的问题，模拟点光源照射的效果，接受阴影等特殊情况。

  3. 数据保存，优化

    最终这些数据会保存为一个序列化的文件，并且提供一些优化手段减少文件大小：从存储来说将TopColor,BottomColor,Specluar编码到了一个float上面，然后在运行时解码用MaterialPropertyBlock传进去就好了。在生成草地阶段，对于缩放旋转可以事先准备一些缩放值和旋转值。生成一个索引列表，然后草地序列化的数据里面只需要存取变换矩阵，索引，encode后的颜色，高光。最后对于整体对序列化的数据进行间隔采样剔除(可以直接成倍的减少，但也会较为明显的影响草地密集的效果)，优化前一个场景的数据多达几M甚至十几M，优化后就只有几K了。
  1. 编辑器配置
    主要是提供操作者自己一些喜欢的操作习惯，快捷键，收藏颜色等。

后续

  1. 我更希望工具能够利用场景的数据进行自动化的生成，之后美术策划需只需要少量的优化工作就能实现整个过程的正常运行。从GDC《幽灵行动：荒野》(原文链接)得到了不少启发，如果能从Houdini和一些dcc软件中能获取到所有环境的数据，那么很多事情将变的非常简单和有趣！
  2. 草地lod并不是很好，参考塞尔达修改一波。
  3. 缺少一些全局的交互效果，燃烧，冰冻，收割，区域生长等。
  4. 之前因为地图不是很大，且草地用四叉树分割加载后，用ComputeShader算Hi-z遮挡剔除发现帧率反而下降,于是暂时没有加上，可能是线程Group设置有问题，之后场景地形确定好新的方案后继续尝试一下。
